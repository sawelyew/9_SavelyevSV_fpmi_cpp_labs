# Аудиторные занятия

Каждая строка текстового файла *classes.txt* содержит информацию об **аудиторном занятиии** (`Lesson`). Там содержится: 
- номер аудитории;
- название дисциплины;
- ФИО преподавателя;
- день недели;
- продолжительность занятия (в минутах).

Все поля разделены точкой с запятой `;`. Гарантируется, что пропусков нет.

Создать классы-наследники: лекция (`Lecture`), семинар (`Seminar`) и лабораторное занятие (`Lab`). Во всех классах переопределить метод  `DisplayInfo`, выводящий информацию о конкретном занятии, и метод `GetType`, возвращающий строковое представление класса (например, в классе `Lab` этот метод вернёт `"Lab"`). 

> У каждой объекта класса должен быть свой **уникальный числовой идентификатор**, не изменяющийся после создания объекта.

Вам необходимо:
1) прочитать содержимое файла в один контейнер(подумайте, что это может быть за контейнер!);
2) после этого предусмотреть выполнение следующих запросов:
    - С помощью **одного цикла** вывести информацию обо всех занятиях.
    - Посчитать количество аудиторных занятий каждого вида.
    - Найти все занятия, проводимые конкретным преподавателем (вводится пользователем).
    - Посчитать суммарную продолжительность всех занятий одного типа.
    - Найти все дисциплины, проводимые в конкретный день недели (вводится пользователем).

Результаты вывести на экран в форме удобной для чтения (для запросов использовать меню). Предусмотреть ситуации, когда пользователь вводит некорректные данные.
