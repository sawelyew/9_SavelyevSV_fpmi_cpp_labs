# Функции

Разработать базовый абстрактный класс `Function` (функция $f(x)$) с тремя методами: `Evaluate` (вычислить значение функции в точке), `Solve` (решить уравнение $f(x) = 0$) и `PrintEquation` (вывести уравнение $f(x) = 0$ на печать). Разработать два производных класса: `LinearFunction` (линейная функция: $f_l(x) = ax + b$) и `QuadraticFunction` (квадратичная функция: $f_q(x) = ax^2 + bx + c$). У каждой объекта класса должен быть свой **уникальный числовой идентификатор**, не изменяющийся после создания объекта.

> Обратите внимание, что $f_q(x) = ax^2 + bx + c$ – квадратичная функция, если $a \neq 0$ (в противном случае такой объект не должен строиться!).

Создать массив, состоящий из объектов обоих типов. При помощи **одного цикла** вывести в файл каждое из получившихся уравнений $f(x) = 0$, а также корни этих уравнений. Если решений нет, то вывести соответствующее сообщение об этом.

В массиве также предусмотреть выполнение следующих запросов:
1) Посчитать количество линейных / квадратичных функций.
1) Найти все уравнения $f(x) = 0$, которые не имеют решений.
2) Найти все функции $f(x)$, у которых $f(x_0) = y_0$, где $x_0, y_0$ вводятся пользователем.
3) Найти все линейные уравнения $f_l(x) = 0$, у которых  b > a, отсортировать их по убыванию коэффициента b.
4) Найти все квадратичные уравнения $f_q(x) = 0$, для которых $f_q(x)$ представляет из себя полный квадрат. Отсортировать их по возрастанию старшего коэффициента.

Результаты вывести на экран в форме удобной для чтения (для запросов использовать меню). Предусмотреть ситуации, когда пользователь вводит некорректные данные.
